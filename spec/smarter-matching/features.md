# TalentLock — Features Specification: Smarter Matching Explanation

## Overview

TalentLock's AI Talent Matching chat (`/ai-match`) currently recommends freelancers but gives employers no explanation for why a freelancer was chosen. Employers see a name and a profile but have to manually verify whether the skills, rate, and availability actually fit their job requirement. This feature adds a structured match explanation card beneath every AI-recommended freelancer — showing exactly which skills matched, which are gaps, whether the rate fits the budget, and whether the freelancer is available for the required start date. The explanation also appears on the freelancer detail page (`/freelancers/:id`) when the employer arrives there from a specific job context.

This is Feature 1 of the AI Enhancements set. It is available on all employer plans (Starter, Growth, Enterprise) and consumes from the existing monthly token quota.

---

## Feature Modules

### 1. Match Explanation Endpoint

A new server-side endpoint `POST /api/ai/match-explanation` accepts a `freelancerId`, an optional `jobRequirementId`, and a `conversationId`. It fetches the freelancer profile and job requirement from the database, builds a structured prompt, calls OpenAI, and returns a parsed JSON explanation with four sections: skills alignment, rate fit, availability fit, and an overall plain-English summary.

Token usage is logged via the existing `logTokenUsage()` utility with feature label `ai_match_explanation` and the active `conversationId`.

---

### 2. Skills Alignment

Shows which skills from the job requirement the freelancer has (matched) and which are missing (gaps). Matched skills render as green chips. Gap skills render as amber chips. If no job requirement is linked to the conversation, skills are assessed against the freelancer's general profile only and gaps cannot be determined — the gaps section is omitted.

---

### 3. Rate Fit

Compares the freelancer's hourly rate against the job requirement's `budgetMin` and `budgetMax`. Returns one of four assessments:

- `within_budget` — freelancer rate falls within the budget range
- `above_budget` — freelancer rate exceeds the budget max
- `below_budget` — freelancer rate is below the budget min (may signal quality concern)
- `unknown` — no budget range set on the job requirement, or no job requirement linked

When assessment is `unknown` the rate fit section is omitted entirely — no placeholder shown.

---

### 4. Availability Fit

Compares the freelancer's `availableFrom` date and `isAvailable` flag against the job requirement's required start date (if set). Returns one of three assessments:

- `available` — freelancer is available on or before the required start date
- `unavailable` — freelancer is booked or their available-from date is after the required start
- `unknown` — no start date on the job requirement, or freelancer has not set availability

---

### 5. Overall Summary

A 2–3 sentence plain-English paragraph generated by the AI summarising the match quality. Always present regardless of whether job requirement context exists.

---

### 6. Match Explanation Card (`<MatchExplanationCard />`)

A reusable React component rendered in two locations:

**Location A — AI Match Chat (`/ai-match`)**
Appears below each freelancer recommendation card in the chat response. Triggered automatically when the AI recommends a freelancer — the employer does not need to click anything.

**Location B — Freelancer Detail Page (`/freelancers/:id`)**
Appears below the freelancer bio when the employer navigates with a `?jobId=` query parameter (i.e. arrived from a specific job's "Find Matches" flow). Not shown when the employer browses the Talent Vault without a job context.

---

## Plan Gating

| Plan | Available |
|---|---|
| `employer_starter` | ✅ |
| `employer_growth` | ✅ |
| `employer_enterprise` | ✅ |

All employer plans have access. Token quota applies — if an employer has exhausted their monthly token quota, the explanation call returns `402 TOKEN_LIMIT` and the card shows a quota-reached state instead of the explanation.

---

## Token Logging

- Feature label: `ai_match_explanation`
- `conversationId` is always passed — populates the `token_usage.conversationId` column added in the ai-enhancements Phase 1 schema migration
- Charged against the employer's monthly token quota via `checkTokenQuota()`

---

## Non-Goals (Out of Scope for This Feature)

- Freelancer-visible match scores or explanations
- Match explanation on the public profile page (`/f/:id`)
- Saving or exporting match explanations
- Comparing multiple freelancers side by side in one explanation
- Match explanation without a freelancer recommendation (e.g. standalone analysis)
- Explanation for why a freelancer was NOT recommended
